# Пользовательские сценарии для интеграционных тестов

Тесты в этом каталоге проверяют связку React (Vite) и Go API в реальном браузере.

## Сценарий A (обязательный): бронирование от выбора типа до успеха

1. Гость открывает страницу записи на тип встречи «Знакомство» (`intro-30`).
2. В календаре выбирает **сегодняшнюю** дату (в пределах 14-дневного окна и рабочих часов бэкенда).
3. В списке слотов выбирает слот со статусом «Свободно».
4. Нажимает «Продолжить».
5. Вводит валидные имя и email, отправляет форму «Подтвердить запись».
6. Видит экран успеха с текстом «Бронь подтверждена».

**Ожидание:** один полный проход гостевого потока без ошибок API и с ожидаемым UI.

## Сценарий B (опциональный): бронь видна владельцу в списке

После успешного сценария A:

1. Открыть страницу «Предстоящие встречи» (`/events`).
2. Убедиться, что в списке есть карточка с именем гостя и контактом, указанными при подтверждении.

**Ожидание:** данные бронирования доступны через API владельца (`/owner/bookings`) и отображаются в UI.
